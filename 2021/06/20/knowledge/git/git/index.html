记录一些常用的Git命令知识。

相关概念

仓库

仓库是一个可以追溯所有历史的文件夹。

  • 其历史的最小单位就是仓库在某个时刻的状态（快照）
  • 通常用SHA-1表示

2、哪些东西可以标识仓库的某个状态？

  • commit对象的SHA-1
  • 分支（branch）
  • HEAD
  • 标签（tag）

3、相关应用
在GitHub上，你可以根据仓库的状态标识，使用url，达到指定状态下的仓库。

commit

仓库从一个状态到另一个状态的这种转移过程，称之为提交，即commit。

在Git中，可以把Commit看成一个对象，它包含id、多个parentId、committer、email等信息

2、注意

  • commit的本质是提交的过程，而不是最终的仓库的状态
  • commit对象是不可变的；git commit --amend底层并不是直接修改commit对象
  • 持续的提交，其本质就是在git仓库维护一个树

3、相关疑问

  • 如何获取tree、commit信息

分支

在git tree中，一个可以移动的指针，其实就是分支，它指向的是仓库的某个状态。

  • HEAD～1：当前节点的父亲节点
  • HEAD^
